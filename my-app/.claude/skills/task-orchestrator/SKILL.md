## Workflow

1. **Before starting a task**: Check dependencies with `checkDependencies`
2. **Start work**: Set status to `in_progress`
3. **During work**: Commit with task ID in message (e.g., `C1: implement feature`)
4. **Complete**: Set status to `done` — blocked dependents auto-unblock
5. **Blocked**: Use `addBlocker` to document why, set status to `blocked`
